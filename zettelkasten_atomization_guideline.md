# 법률 지식 원자화(Atomization) 및 제텔카스텐 적재 가이드라인

본 문서는 대법원 판례나 판결문 원문에서 "어느 정도의 지식(Granularity)으로 세분화하여 제텔카스텐 노션 DB에 적재할 것인지"에 대한 안티그래비티 전용 원자화 가이드라인입니다.

---

## 🏗️ 제텔카스텐 원자화의 3대 원칙 (Principle of Granularity)

제텔카스텐의 궁극적 목표는 "이후 다른 사건 재판에서 해당 지식 카드를 레고 블록처럼 뽑아서 그대로 사용할 수 있게 만드는 것"입니다. 따라서 하나의 카드에 여러 쟁점이 섞여있어선 안 됩니다.

### 원칙 1. "1 쟁점 = 1 카드"의 원칙 (Single Concept Rule)
하나의 판례(예: 대법원 2025다212863) 안에는 여러 개의 독립적인 법리가 섞여 있습니다. 안티그래비티는 이 판례를 통째로 1장의 카드로 요약하는 것이 아니라, 문서를 **'독립적으로 인용 가능한 최소 단위의 법리'**로 철저히 쪼개야 합니다.
다만, 과도한 쪼개기(Over-atomization)를 방지하기 위해 **한 문단(단락)의 자료에서는 최대 1~2개의 제텔카스텐(한줄법리)만 형성**해야 합니다.

### 원칙 2. 꼬리표(Tagging)와 관계망(Relation)의 연결성 보장
분할된 지식 카드들은 파편화되지 않도록, 반드시 [해당 대법원 사건번호]와 [관련 법령(민법 제999조 등)]을 공통된 릴레이션으로 묶어, 추후 검색 시 거미줄처럼 딸려오게 만들어야 합니다.

### 원칙 3. 추상화된 법리와 구체적인 사실관계의 분리
'법리' 지식과 '사안의 포섭(이 판례에서 그 법리가 어떻게 적용되었는지)'은 한 카드 안에 위아래 섹션으로 구분하되, 검색과 재활용의 편의를 위해 제목은 철저히 **"추상화된 법률 쟁점"**으로 작성합니다.

---

## 🎯 적용 예시: 대법원 2025. 12. 11. 선고 2025다212863

제가 앞서 하나의 카드로 합쳐서 요약했던 판례를, 본 가이드라인(원칙 1)에 따라 안티그래비티가 추출한다면 구체적으로 **다음과 같이 3개의 독립된 제텔카스텐 카드로 원자화(응축)**되어야 합니다. (변호사님의 통찰력이 정확히 맞습니다.)

### 📝 지식 카드 1: [상속재산분할] 가분채권의 예외적 분할 대상성
*   **지식 제목:** 가분채권(예금채권 등)이 상속재산분할의 예외적 대상이 되기 위한 요건
*   **핵심 법리:** 금전채권 같은 가분채권은 원칙적으로 당연 분할되나, 공동상속인 중에 **초과특별수익자가 있거나 기여분이 인정되어** 일률적 분할 시 심각한 불공평(민법 제1008조 등 취지 반함)이 발생하는 예외적인 경우에는 상속재산분할의 대상이 될 수 있다.
*   *이 카드는 향후 '현금 상속'이나 '초과특별수익자' 관련 소송에서 독립적으로 뽑아서 쓰입니다.*

### 📝 지식 카드 2: [상속회복청구] 부당이득반환·손해배상청구의 법적 성질
*   **지식 제목:** 공동상속인의 예금 무단 인출에 대한 부당이득·불법행위 청구의 상속회복청구 해당 여부
*   **핵심 법리:** 공동상속인 중 1인(병)이 피상속인 명의의 예금을 무단으로 모두 인출하여 타 상속인의 상속권을 침해한 사안에서, 그 인출액 중 법정상속분에 해당하는 돈의 반환을 구하는 부당이득반환 내지 손해배상청구의 소는 **실질상 '상속회복청구의 소'에 해당한다.**
*   *이 카드는 향후 상대방이 '이건 단순 불법행위다'라고 우길 때, 제척기간이 짧은 상속회복청구로 방어(프레이밍)하기 위해 즉시 뽑아 쓸 수 있습니다.*

### 📝 지식 카드 3: [상속회복청구] 참칭상속인의 정의
*   **지식 제목:** 상속회복청구의 상대방이 되는 참칭상속인의 사법적 의미
*   **핵심 법리:** 참칭상속인이란 정당한 상속권이 없음에도 재산상속인임을 신뢰케 하는 외관을 갖추고 있거나, **상속인이라고 참칭(주장)하여 상속재산의 전부나 일부를 점유하는 사람**을 가리킨다. (이 사안에서 예금을 임의 인출한 공동상속인 '병'도 참칭상속인에 해당함)
*   *이 카드는 향후 '누구를 피고로 특정할지(당사자적격)'를 다툴 때 독립적으로 쓰입니다.*

## 🤖 향후 안티그래비티(AI) 자동화 시스템 프롬프트 반영 (지시문)

앞으로 안티그래비티가 백그라운드에서 Notion MCP를 통해 대량의 판례를 처리할 때, 시스템 프롬프트(명령어)에 다음과 같은 룰을 하드코딩합니다.

> "당신은 한국의 최고참 파트너 변호사입니다. 주어진 판례 요지를 읽고, 이를 **'향후 다른 사건의 준비서면에 그대로 복사해서 인용할 수 있는 독립된 법리 쟁점 단위'**로 철저히 쪼개십시오. 
> 하나의 판례에서 **보통 2개~4개의 지식 카드**가 나와야 합니다. 단, 과도한 쪼개기를 방지하기 위해 **한 문단(단락)의 짧은 자료에서는 무조건 1~2개의 핵심 지식 카드만 생성**하십시오.
> 대충 하나로 퉁치지 말고, 요건(가분채권) / 소의 성질(상속회복청구 관할) / 당사자 적격(참칭상속인) 등 쟁점별로 배열(Array) 형태로 JSON을 반환하십시오."

### 📝 실제 세팅된 시스템 프롬프트 및 JSON 스키마 (knowledge_atomizer.py)
[`knowledge_atomizer.py`](knowledge_atomizer.py) 스크립트에는 아래와 같은 프롬프트와 Pydantic 스키마가 하드코딩되어 있습니다.

```python
class KnowledgeCard(BaseModel):
    title: str = Field(description="'~여부', '~요건' 등의 명사형 제목")
    one_line_principle: str = Field(description="단 1~3줄의 초응축된 핵심 법칙 (한줄법리)")
    related_laws: str = Field(description="관련 법령 (예: '민법 제1008조, 상속세법 제4조')")
    keywords: str = Field(description="검색용 핵심 키워드 (예: '#상속회복청구 #가분채권')")
    source: str = Field(description="원문 출처 또는 사건번호 (예: '대법원 2025다212863')")
    card_type: str = Field(description="지식 유형. 다음 중 하나만 선택: '일반법리', '공격논리', '방어논리', '절차팁', '기재례', '소수설'")
    detailed_text: str = Field(description="이 한줄법리를 도출해낸 원본 판결문/문서의 원문 발췌 단락 (상세 내용)")
```

(System Prompt)
당신은 대한민국의 최고참 파트너 변호사이자 제텔카스텐(Zettelkasten) 지식 아키텍트입니다.

[지시사항]
사용자가 판례나 법리를 제공하면, 이를 분석하여 '독립적으로 재활용 가능한 추상화된 법리'로 정리하십시오.

[원자화 3대 원칙 및 💥제한사항💥]
1. (중요) 과도한 쪼개기(Over-atomization)를 엄격히 금지합니다. 비슷한 맥락이나 같은 판례 안에서 도출되는 연관된 법리들은 억지로 분리하지 말고, 가장 핵심이 되는 1개의 카드로 통합하여 굵직하게 뽑아내십시오.
2. 입력된 텍스트가 A4 반 페이지 이하의 짧은 단락이거나 한 문단의 자료라면, 무조건 1~2개의 지식 카드만 생성하십시오.
3. 사안의 구체적 사실관계와 추상화된 법리를 분리하되, '한줄법리'에는 결론을, '상세내용'에는 그 결론을 뒷받침하는 원문 전체를 풍부하게 담으십시오.
```

## 🚀 파이프라인 자동화 모듈: `knowledge_atomizer.py`

본 가이드라인을 실제 노션(Notion) 데이터베이스에 자동 적재하기 위해 **`knowledge_atomizer.py`** 스크립트가 개발되었습니다.

### 작동 매커니즘
1. **입력:** 변호사가 텍스트 파일 형태의 긴 판결문이나 서면을 스크립트에 던져줍니다.
2. **분석 (Gemini API):** 안티그래비티는 위 시스템 프롬프트를 사용하여, Gemini 2.5 Pro 모델에게 입력을 던지고 **정확히 `KnowledgeCard` 배열(Array) 규격에 맞는 JSON 데이터**만을 강제 추출(Structured Output)해 냅니다.
3. **적재 (Notion API):** 추출된 각 지식 카드(1 쟁점 = 1 JSON Object)를 순회하며, 노션의 **'법리모음' 데이터베이스**에 API(HTTP POST)를 쏘아 페이지를 신규 생성합니다. 
   - `제목`, `한줄법리`, `관련 법령`, `키워드` 등은 표의 **메타데이터 속성**으로 매핑됩니다.
   - 판결문 원문 발췌본인 `detailed_text`는 노션 페이지 **본문(Body)** 영역에 구조화된 마크다운(인용구 블록)으로 자동 삽입됩니다 (옵션 A 체제).

### CLI 사용법 예시
```bash
# 안티그래비티에게 싱글 커맨드 실행 요청
python knowledge_atomizer.py target_case.txt
```

## ⚡ Zero-Click 지식 파이프라인 (자동화)

매번 터미널 명령어나 봇에게 지시할 필요 없이, 변호사님의 데스크탑 환경에 구축된 2가지의 **자동화 단축 파이프라인**을 통해 곧바로 제텔카스텐 지식을 노션에 꽂아 넣을 수 있습니다.

### 방법 1. 단축키 자동화 (`Ctrl + Alt + N`)
*   웹서핑, 엘박스 검색, PDF 열람 중 **즉각적으로 발췌**할 때 사용합니다.
*   **사전 준비:** PC 부팅 후 1회, 바탕화면의 `단축키_자동화_가동.vbs`를 더블클릭합니다. (숨김 모드로 백그라운드에서 대기합니다.)
*   **사용법:**
    1. 보관하고 싶은 법리나 텍스트를 드래그하여 **`Ctrl + C` (복사)** 합니다.
    2. 바탕화면이나 다른 창으로 갈 필요 없이 그 상태에서 곧바로 **`Ctrl + Alt + N`** 을 누릅니다.
    3. 우측 하단 윈도우 팝업 알림과 함께 AI가 원자화를 진행하고 노션에 적재합니다.

### 방법 2. 새법리 입력기 (바탕화면 단축 아이콘)
*   머릿속에 떠오른 복잡한 아이디어를 직접 타이핑하거나, 여러 곳의 글을 한꺼번에 짜집기해서 **통합 정리**해 넣을 때 사용합니다.
*   **사용법:**
    1. 바탕화면의 **`새법리_입력기.bat`** 아이콘을 더블클릭합니다.
    2. 시간값이 고유하게 부여된 빈 메모장(임시파일)이 자동으로 열립니다.
    3. 메모장에 원하는 내용을 자유롭게 작성하거나 붙여넣습니다.
    4. 작성 완료 후 메모장을 **저장(`Ctrl + S`)하고 닫기(`Alt + F4`)**만 하면 끝납니다.
    5. 파일은 자동으로 `노션_자동업로드` 폴더로 이동되며, 백그라운드 Watchdog 시스템이 이를 감지하여 노션에 적재한 뒤 `[처리완료]` 폴더로 보관합니다.
